# 세로 텍스트 방향 처리 비교

## 배경

`useMarketGuideViewer`에서 매장 영역(rect)이 가로보다 세로가 길 때, 매장명 텍스트를 세로로 표시해야 했음. 두 가지 방식을 시도함.

---

## 방식 1: `transform: rotate(-90)` (처음 제시한 방법)

```typescript
storeGruops.append('text')
  .attr('x', (store) => store.x + store.width / 2)
  .attr('y', (store) => store.y + store.height / 2)
  .attr('text-anchor', 'middle')
  .attr('dominant-baseline', 'middle')
  .attr('font-size', 6)
  .attr('transform', (store) =>
    store.height > store.width
      ? `rotate(-90, ${store.x + store.width / 2}, ${store.y + store.height / 2})`
      : ''
  )
  .text((store) => store.storeName ?? '');
```

**동작:** 가로 텍스트를 통째로 -90도 회전시켜 세로처럼 보이게 함. 글자가 옆으로 누운 형태(예: "시흥상회"를 회전시킨 모양)로 표시됨.

**특징**
- 회전 기준점(`rotate(-90, x, y)`)을 매번 직접 계산해야 함
- 글자가 한 줄로 누워서 회전되는 것이라, 한글을 위에서 아래로 한 글자씩 쌓아 읽는 형태는 아님
- SVG `transform` 속성 사용 → 호환성 넓음

---

## 방식 2: `writing-mode: tb` (사용자 수정)

```typescript
storeGruops.append('text')
  .attr('x', (store) => store.x + store.width / 2)
  .attr('y', (store) => store.y + store.height / 2)
  .attr('text-anchor', 'middle')
  .attr('dominant-baseline', 'middle')
  .attr('font-size', 6)
  .attr('writing-mode', (store) =>
    store.height > store.width
    ? "tb" : ''
  )
  .text((store) => store.storeName ?? '');
```

**동작:** CSS/SVG의 텍스트 흐름 방향 자체를 `tb`(top-to-bottom)로 전환. 회전 없이 글자가 위에서 아래로 한 글자씩 쌓여서 표시됨 (예: "시 / 흥 / 상 / 회").

**특징**
- 좌표 계산이나 회전 기준점 계산 불필요 — `x`, `y` 그대로 사용
- 한글 세로쓰기 형태로 자연스럽게 표시됨
- 별도 회전 로직, `tspan` 분리 로직 없이 속성 하나로 해결되어 코드가 더 간결함

---

## 비교 결과

| | rotate(-90) | writing-mode: tb |
|---|---|---|
| 코드 복잡도 | 회전 기준점 계산 필요 | 속성 값만 분기, 좌표 계산 불필요 |
| 텍스트 표현 | 가로 문장을 통째로 회전 (누운 형태) | 글자가 위→아래로 쌓이는 세로쓰기 |
| 가독성 | 회전된 가로 문장 형태 | 한글 세로쓰기에 더 자연스러움 |

**결론:** `writing-mode: tb` 방식이 더 간단하고 세로 매장명 표시 목적에 더 적합해서 최종 적용함.
