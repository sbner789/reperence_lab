# 2026-07-02 Decision

## Sortable 컴포넌트: 드래그 앤 드롭 반대 방향 버그 수정

**요구사항:** 리스트 순서를 드래그로 바꿀 때, 처음 아이템을 다른 곳으로 옮기면 정상 동작하지만 반대 방향(다른 아이템 → 첫 아이템)으로 옮기면 동작하지 않음

**수행 내용:**
- 원인: `handleDrop`의 `if (!draggedItem || !changed) return;`에서 `changed`가 `orderIndex === 0`인 아이템일 때 숫자 `0`이 falsy로 취급되어 조건문이 항상 `return`됨
- `draggedItemRaw === ''`(빈 문자열) 여부로 판단하도록 조건을 명시적으로 수정해 `0` falsy 문제 해결

## Sortable 컴포넌트: 실제 배열 재정렬 로직 구현

**요구사항:** `splice`로 위치는 바뀌는데 내부 `orderIndex` 값이 갱신되지 않고 에러가 발생하는 문제 해결. 위치와 내부 값이 함께 갱신되길 원함

**수행 내용:**
- 원인 1: `useState<[]>([])`로 선언되어 TS가 배열 원소 타입을 `never`로 추론 → `el.orderIndex` 접근 시 타입 에러
- 원인 2: `changed`가 실제로는 숫자(`orderIndex`)인데 타입은 객체(`T`)로 선언되어 있었고, `dragged[0].orderInde`는 오타로 항상 매칭 실패
- 드래그 시작 시점의 **배열 인덱스**를 식별자로 사용하도록 변경, `splice`로 실제 위치 이동 후 `newItems.map((item, index) => ({ ...item, [keyName]: index }))`로 전체 아이템의 순서 필드를 위치 기준으로 재계산
- `Props<T>`를 `SortableConfig<T>`(`list`, `keyName`, `content`)로 재정의해 `initialConfig`가 아이템 타입 `T`를 기준으로 타입 체크되도록 정리
- 미사용 `useRef` import 제거 (`noUnusedLocals: true`로 인해 컴파일 에러 원인이었음)

## Sortable 컴포넌트: 드래그 중 자동 스크롤 추가

**요구사항:** 스크롤이 있는 리스트에서 맨 위 ↔ 맨 아래로 드래그할 때 스크롤이 따라가지 않아 불편함

**수행 내용:**
- 스크롤 컨테이너(`.sortable`)에 `scrollContainerRef` 연결
- `handleDragOver`에서 `e.clientY`와 컨테이너 상/하단 경계 간 거리를 계산, 60px 이내로 가까워지면 `scrollTop`을 20px씩 조정해 양방향 자동 스크롤 구현

## 리렌더링 최적화(useMemo) 검토 → 보류

**요구사항:** 순서 변경 시 값이 바뀌지 않은 아이템까지 리렌더링되는 것을 `useMemo`로 최적화할 수 있는지 검토

**수행 내용:**
- `useMemo`는 계산값 캐싱용이지 개별 리스트 아이템의 리렌더링을 막는 훅이 아님을 확인
- 현재 재정렬 로직이 순서가 안 바뀐 아이템까지 포함해 배열 전체를 매번 새 객체로 생성하고 있어, `React.memo`를 적용해도 효과가 없는 구조였음을 확인
- 다만 이 컴포넌트는 "표시할 테이블 컬럼 순서/노출 여부를 설정하는 UI"로, 항목 수가 컬럼 개수(현재 16개)에 고정되고 데이터 row 수와 무관하게 늘어나지 않는 구조 → 대량 리스트 최적화 케이스와 성격이 다르다고 판단, **최적화 작업은 보류**

## 컬럼 표시/숨김(`isVisible`) 토글 기능

**요구사항:** `isVisible: false`로 설정하면 실제로 표시되는 리스트(테이블)에서 해당 컬럼이 보이지 않도록 구현. 단, 실제 표시 리스트(테이블)는 개발자가 별도로 개발 예정이므로 이번 범위는 **Sortable의 토글 버튼 동작까지만**

**수행 내용:**
- `useSortable`에 범용 `updateItem(index, patch: Partial<T>)` 추가 — 특정 필드명에 종속되지 않고 배열의 특정 위치 아이템을 부분 업데이트
- 토글 영역에 `onClick={() => updateItem(item.orderIndex, { isVisible: !item.isVisible })}` 연결, `toggle_filter_active` / `toggle_filter_deactivate` 클래스로 상태 표시
- 토글 스위치 원(`toggle_knob`)의 이동 거리를 하드코딩된 `30px` 대신 CSS 변수 + `calc()`로 계산하도록 변경
  ```css
  --toggle-track-width: 70px;
  --toggle-track-padding: 5px;
  --toggle-knob-size: 30px;
  transform: translateX(calc(var(--toggle-track-width) - (var(--toggle-track-padding) * 2) - var(--toggle-knob-size)));
  ```
  → 트랙 너비/패딩/원 크기 중 하나가 바뀌어도 자동으로 재계산됨
- `isVisible`에 따라 드래그 이벤트(소스: `draggable`, 타겟: `onDragOver`/`onDrop`)를 막는 로직을 추가했으나, **개발자가 보류 결정 — 현재 코드에는 `draggable={item.isVisible}`이 주석 처리된 채로 남아있고 `draggable`(항상 true)로 되돌아간 상태**. 재검토 시 이어서 진행 필요
