# 2026-06-22 Decision

## MarketGuideViewer 텍스트 위치 버그 수정

**요구사항:** `useMarketGuideViewer`에서 매장명 텍스트가 원하는 위치에 표시되지 않는 문제 해결

**수행 내용:**
- 원인: `store.x as number + store.width as number / 2`에서 TS `as` 연산자가 `+`, `/`보다 낮은 우선순위로 파싱되어 `(store.x + store.width) / 2`로 컴파일됨 (의도한 `store.x + store.width/2` 중심점 계산이 아님)
- 수정: `(store.x as number) + (store.width as number) / 2` 형태로 괄호를 명시해 우선순위 문제 해결
- `y` 좌표도 동일하게 수정

## MarketGuideViewer d3 Nested Data Join 고도화

**요구사항:** sector → section → store로 이어지는 3중 `.map()` 중첩 구조를 d3 패턴으로 더 깔끔하게 풀어낼 수 있는지 확인하고 고도화

**수행 내용:**
- d3의 Nested Data Join 패턴 적용: 계층별로 `selectAll().data().join()`을 반복하는 구조로 전환, 수동 `.append('g')` 중첩 루프 제거
- 매장명 매칭 로직을 `storeData` 전체 순회(O(n×m))에서 `useMemo` 기반 `Map` 조회(O(1))로 개선
- `useMarketGuide.tsx`에 `StoreShape`, `MarketGuideSection`, `MarketGuideSector`, `StoreNameEntry` 타입을 `export`하여 `useMarketGuideViewer`와 타입 공유 (`any` 미사용)
- 상세 비교 및 접근 내용은 `docs/suggestion/MarketGuideViewer-d3-Nested-Data-Join-고도화.md`에 기록
